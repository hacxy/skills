# fit-docs

一个用于生成需求文档（PRD）、技术设计文档（TDD）和架构设计文档的 skill，核心目标是**完全满足用户需求**。

与其他文档工具的最大区别：**以产品经理的视角分析需求** —— 在动笔之前，先深入挖掘需求的目的、价值、用户体验和产品闭环，再主动澄清所有模糊点。最终产出的文档不仅精确匹配你的需求，还能揭示你未曾想到的洞察。

→ [English](README.md)

## 功能

- **产品经理视角分析** —— 深入挖掘需求目的、价值主张、用户体验和产品闭环
- 生成 **PRD**、**TDD** 和**架构设计文档**，格式为 Markdown
- 在撰写任何章节之前，主动提问以填补信息缺口
- 在生成完整文档前，先展示提纲供你确认
- 将文件保存至 `docs/` 下的合理目录结构中
- **语言自动匹配**：你用中文描述需求，就输出中文文档
- 持续迭代，直到你满意为止

## 触发条件

当你说以下类似的话时，该 skill 会自动激活：
- "帮我写一个 XXX 功能的 PRD"
- "生成技术设计文档"
- "写一份架构设计文档"
- "帮我把这个需求文档化"
- "写一个功能规格说明"

## 安装

```bash
npx skills add <owner/repo> --skill fit-docs
```

## 输出目录结构

```
docs/
├── requirements/     # 需求文档（PRD）
├── technical/        # 技术设计文档（TDD）
└── architecture/     # 架构设计文档
```
